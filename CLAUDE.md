# Project CLAUDE.md - Tomorrow Night Darkly

## Project Overview

A dark color theme ("A Simply Dark Theme") for Chrome, GitHub, and as a React-based preview site.

## Tech Stack

- **Language:** TypeScript (ESM)
- **Frontend:** React 19, react-icons, react-cool-img
- **Backend:** Koa with koa-static, @koa/cors
- **Bundler:** Webpack 5 with Babel (TS + React presets)
- **Styling:** Sass (SCSS), the-new-css-reset
- **Linting:** mikey-pro (ESLint + Prettier + Stylelint)
- **Deployment:** AWS S3 + CloudFront (via GitHub Actions)
- **CI/CD:** GitHub Actions (deploy, SonarCloud, Dependabot auto-merge)

## Architecture

```
chrome/
  theme/
    manifest.json       # Chrome theme manifest (Manifest V3)
  promo/                # Chrome Web Store promotional assets
client/
  src/
    index.tsx           # React entry point
    components/App.tsx  # Main App component
    styles.scss         # Full SCSS stylesheet
github/
  github-theme-tomorrow-night-darkly.css  # GitHub custom CSS theme
public/
  index.html            # HTML shell (includes GTM)
  dist/bundle.js        # Webpack output
  assets/               # Favicon, SVG logo, social image
icon/                   # Theme icons (SVG + PNG)
.github/
  workflows/
    deploy.yml          # S3 deploy + CloudFront invalidation
    sonar.yml           # SonarCloud analysis
    auto-merge.yml      # Dependabot auto-merge
app.ts                  # Koa server (port 8080)
webpack.config.ts       # Webpack configuration
tsconfig.json           # TypeScript config
eslint.config.js        # ESLint flat config (delegates to mikey-pro)
```

## Commands

```bash
npm run build:dev       # Webpack dev build (watch mode)
npm run build:prod      # Webpack production build
npm run start:client    # Start Koa server (ts-node app.ts)
npm run test:client     # Start server with nodemon (hot reload)
```

## Conventions

- ESM (`"type": "module"`)
- Chrome theme uses Manifest V3
- Prettier via `mikey-pro/prettier`
- Stylelint via `mikey-pro/stylelint`
- Conventional commits

## Deployment

- Push to `main` triggers GitHub Actions deploy workflow
- Builds with `npm run build:prod`, syncs `public/` to S3, invalidates CloudFront
- SonarCloud runs on push to main and on PRs
- Dependabot auto-merges its own PRs with squash

## Color Palette

Core colors used across Chrome theme, GitHub CSS, and preview site:
- `#202020` -- primary background (gray)
- `#181818` -- darker background (toolbar/NTP)
- `#b294bb` / `#b392bd` -- accent purple/pink (tabs, links, accents)
- `#c6c6c6` -- light gray text

## Common Mistakes

- Chrome theme version in `chrome/theme/manifest.json` and `package.json` must stay in sync
- The `public/dist/` directory is gitignored -- it is generated by webpack
- `ts-node` uses a CommonJS override in `tsconfig.json` (`"ts-node"` section) since Koa runs in Node, not the browser
- Webpack config has `jsxPragma: 'h'` in the TypeScript preset -- this is intentional for the Babel pipeline
